# 📊 HPE CCA Data Analysis Automation

Этот проект предназначен для автоматизации обработки и анализа отчетов **HPE Sell Track** по региону **Caucasus & Central Asia (CCA-8)**. 

Инструментарий позволяет быстро фильтровать огромные корпоративные отчеты, извлекать актуальные данные по ключевым странам и подготавливать их для использования в **NotebookLM** или других аналитических системах.

---

## 🌍 Целевой регион (CCA-8)
Скрипты автоматически фильтруют данные по следующим странам:
*   🇰🇿 Казахстан
*   🇺🇿 Узбекистан
*   🇦🇿 Азербайджан
*   🇬🇪 Грузия
*   🇦🇲 Армения
*   🇹🇯 Таджикистан
*   🇰🇬 Кыргызстан
*   🇹🇲 Туркменистан

---

## 🚀 Основные возможности
1.  **Авто-обнаружение обновлений**: Скрипт сам находит последний загруженный `.xlsx` файл и извлекает дату его обновления (Refresh Date).
2.  **Умная фильтрация**: Сбор данных из всех листов (Compute, Networking, HC, Competencies) с сохранением контекста.
3.  **Сравнительный анализ**: При каждом обновлении система выводит человекочитаемый отчет о том, что именно изменилось (сколько строк добавилось и по каким странам/категориям).
4.  **Оптимизация для NotebookLM**: Выходной CSV-файл структурирован так, чтобы ИИ максимально точно понимал бизнес-контекст каждой записи.

---

## 🛠 Инструкция по использованию

### 1. Добавление новых данных
Просто поместите ваш исходный файл `.xlsx` в папку `data/raw/`. 
*Рекомендуется формат имени: `HPE_Source_YYYY-MM-DD.xlsx`.*

### 2. Запуск обработки
Выполните команду в терминале:
```bash
/Users/anonimouskz/anaconda3/bin/python3 scripts/sync_and_analyze.py
```

### 3. Получение результата
Готовый файл для анализа появится в:
`data/processed/HPE_CCA_YYYY-MM-DD.csv`

## 📂 Структура проекта
*   `data/raw/`: Исходные отчеты (игнорируются Git).
*   `data/processed/`: Отфильтрованные CSV-файлы для анализа (игнорируются Git).
*   `scripts/`: Логика обработки данных на Python.
*   `GEMINI.md`: Инструкции для ИИ-ассистентов по поддержке проекта.
